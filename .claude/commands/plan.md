# /plan — Planejar antes de implementar

Antes de escrever qualquer código, execute este fluxo completo:

1. Leia `CLAUDE.md` na raiz do projeto
2. Leia `specs/poc-tecnica.md` inteiro
3. Identifique qual fase está em andamento (H2 → H1 → H3) com base nos checkboxes de specs/poc-tecnica.md
4. Liste os arquivos que precisam ser criados ou modificados para a próxima tarefa
5. Apresente o plano em formato de checklist antes de executar qualquer ação
6. Aguarde confirmação do usuário antes de sair do modo de planejamento

**Nunca inicie a implementação sem confirmar o plano com o usuário.**
